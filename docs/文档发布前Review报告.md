## 一、严重问题（发布前必须修复）

### 🔴 C1：费用控制方案中 DeepSeek 缓存命中价格错误（30 倍偏差）

**位置**：`API中转站_费用控制方案设计.md` 第 110-111 行

```python
# ❌ 当前错误值
"deepseek-v4-pro":   {"input": 3.0, "output": 6.0, "cache_hit": 0.75},
"deepseek-v4-flash": {"input": 1.0, "output": 2.0, "cache_hit": 0.25},
```

```python
# ✅ 正确值（来自 Review 文档和 DeepSeek 官方定价）
"deepseek-v4-pro":   {"input": 3.00, "output": 6.00, "cache_hit": 0.025},
"deepseek-v4-flash": {"input": 1.00, "output": 2.00, "cache_hit": 0.02},
```

**影响**：缓存命中费用被放大 30 倍（V4-Pro：¥0.025→¥0.75，V4-Flash：¥0.02→¥0.25），会导致费用估算严重失真。
