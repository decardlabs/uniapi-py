# UniAPI 系统配置指南

## 一、配置体系概述

UniAPI 的系统配置分两层：

| 层级 | 存储方式 | 配置内容 | 修改方式 |
|------|----------|----------|----------|
| **环境变量** | `.env` 文件 | 数据库路径、端口、上游 API Key、Redis 等 | 编辑 `.env` + 重启服务 |
| **系统选项** | SQLite 数据库 `options` 表 | 系统名称、Logo、注册开关、验证码等 | API 修改，即时生效 |

**环境变量配置** 详见 [部署方案.md](部署方案.md)。

**系统选项** 通过管理 API 操作，**修改后无需重启服务**，前端自动感知。

### 修改系统选项

通过管理 API（需要 root 权限）：

```bash
# 修改系统名称
curl -X PUT 'https://your-domain/api/option/' \
  -H 'Authorization: Bearer <access_token>' \
  -H 'Content-Type: application/json' \
  -d '{"key": "SystemName", "value": "我的AI网关"}'

# 关闭注册
curl -X PUT 'https://your-domain/api/option/' \
  -H 'Authorization: Bearer <access_token>' \
  -H 'Content-Type: application/json' \
  -d '{"key": "RegisterEnabled", "value": "false"}'

# 开启邮箱验证
curl -X PUT 'https://your-domain/api/option/' \
  -H 'Authorization: Bearer <access_token>' \
  -H 'Content-Type: application/json' \
  -d '{"key": "EmailVerificationEnabled", "value": "true"}'
```

查看当前所有选项：

```bash
curl 'https://your-domain/api/option/' \
  -H 'Authorization: Bearer <access_token>'
```

## 三、新用户注册配置

### 3.1 注册控制开关体系

```
RegisterEnabled (总开关)
  ├── true → 显示注册入口
  │     ├── PasswordRegisterEnabled=true  → 用户名/密码注册可用
  │     ├── PasswordRegisterEnabled=false → 隐藏密码注册表单
  │     ├── EmailVerificationEnabled=true  → 注册需邮箱验证码
  │     ├── EmailVerificationEnabled=false → 注册无需邮箱验证
  │     ├── turnstile_check (前端硬编码)    → 开启 Cloudflare Turnstile
  │     └── github_oauth (前端硬编码)       → 开启 GitHub 三方登录
  └── false → 隐藏注册入口，仅管理员可创建用户
```

> ⚠️ **注意**：`turnstile_check`、`github_oauth` 目前在前端硬编码返回 `false`，如需开启需修改代码（见下文）。

### 3.2 注册流程（默认配置，无需邮箱验证）

```
用户填写表单
  ├── 用户名 (必填，≥3字符)
  ├── 密码 (必填，≥8字符)
  ├── 确认密码 (必填)
  ├── 邮箱 (选填，前端校验格式)
  ├── 验证码 (选填，前端占位，后端未实现)
  └── 邀请码 (选填)
        ↓
发送 POST /api/user/register
        ↓
后端校验 → 用户名唯一性、密码长度
        ↓
创建用户 (status=1 正常，role=1 普通用户)
  ├── 初始额度：1,000,000 tokens
  ├── 所属分组：default
  └── 自动生成默认令牌
        ↓
创建会话 → 自动登录 → 跳转到首页
```

### 3.3 注册 API 说明

**POST `/api/user/register`**

请求体：
```json
{
    "username": "newuser",
    "password": "password123",
    "display_name": "新用户",
    "email": "user@example.com",
    "verification_code": "123456",
    "aff_code": "INVITE2024"
}
```

**说明：**
- `display_name`、`email`、`verification_code`、`aff_code` 均为可选字段
- 邮箱验证码功能 (`verification_code`) 后端暂未实现（见第五章）
- `aff_code` 为邀请码，系统会记录邀请关系

## 六、Cloudflare Turnstile 配置

### 6.1 获取密钥

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 进入 **Turnstile** → **添加站点**
3. 站点名称: `UniAPI`
4. 域名: `your-domain.com`
5. 获取 **Site Key** 和 **Secret Key**

### 6.2 配置方式

**环境变量 `.env`（密钥）：**
```bash
TURNSTILE_SECRET_KEY=0x4AAAAAA...
```

**系统选项（非密钥，通过 `PUT /api/option/` 设置）：**
```bash
# 开启 Turnstile
curl -X PUT 'https://your-domain/api/option/' \
  -H 'Authorization: Bearer <access_token>' \
  -d '{"key": "TurnstileCheckEnabled", "value": "true"}'

# 设置 Site Key
curl -X PUT 'https://your-domain/api/option/' \
  -H 'Authorization: Bearer <access_token>' \
  -d '{"key": "TurnstileSiteKey", "value": "0x4AAAAAA..."}'
```

### 6.3 后端验证（已实现）

添加 Turnstile Token 验证（在注册和发验证码端点中）：

```python
async def verify_turnstile(token: str) -> bool:
    import httpx
    secret = settings.turnstile_secret_key
    if not secret:
        return True  # 未配置时放行
    async with httpx.AsyncClient() as client:
        resp = await client.post(
            "https://challenges.cloudflare.com/turnstile/v0/siteverify",
            data={"secret": secret, "response": token},
        )
        result = resp.json()
        return result.get("success", False)
```

## 七、GitHub OAuth 配置（已实现）

### 7.1 创建 GitHub OAuth App

1. 登录 GitHub → **Settings** → **Developer settings** → **OAuth Apps** → **New OAuth App**
2. 填写信息：
   - **Application name**: `UniAPI`
   - **Homepage URL**: `https://your-domain.com`
   - **Authorization callback URL**: `https://your-domain.com/api/oauth/github`
3. 获取 **Client ID** 和生成 **Client Secret**

### 7.2 配置方式

**环境变量 `.env`（密钥）：**
```bash
GITHUB_CLIENT_SECRET=xxxxxxxxxxxxxxxxxx
```

**系统选项（非密钥，通过 `PUT /api/option/` 设置）：**
```bash
# 开启 GitHub OAuth
curl -X PUT 'https://your-domain/api/option/' \
  -H 'Authorization: Bearer <access_token>' \
  -d '{"key": "GitHubOAuthEnabled", "value": "true"}'

# 设置 Client ID
curl -X PUT 'https://your-domain/api/option/' \
  -H 'Authorization: Bearer <access_token>' \
  -d '{"key": "GitHubClientId", "value": "Iv1.xxxxxxxxxx"}'
```

### 7.3 可用端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/oauth/state` | GET | 获取 CSRF state token |
| `/api/oauth/github?code=xxx&state=yyy` | GET | GitHub OAuth 回调 |

### 7.4 注册流程

1. 前端调用 `GET /api/oauth/state` 获取 state
2. 用户跳转 GitHub 授权页面
3. GitHub 回调到 `GET /api/oauth/github`
4. 后端验证 state → code 换 token → 获取用户信息 → 查找/创建用户
5. 返回登录响应（含 session cookie）

## 八、邀请码系统

### 8.1 工作机制

- 用户在注册时可填入 `aff_code`（邀请码/推广码）
- 后端 `register_user()` 接受 `aff_code` 参数
- 邀请码对应推广用户的 `aff_code` 字段
- 数据库 `User` 表有 `aff_code`（自己的推广码）和 `inviter_id`（邀请人 ID）字段

### 8.2 后台设置推广码

管理员可在后台为用户设置推广码，或通过 API：

```bash
curl -X PUT 'https://your-domain/api/user/' \
  -H 'Authorization: Bearer <access_token>' \
  -H 'Content-Type: application/json' \
  -d '{"id": 1, "group": "default"}'
```

> ⚠️ 当前注册接口 `register_user()` **接收** `aff_code` 但**未实际处理邀请关系**（设置了 `inviter_id` 后才算关联）。如需完整邀请返利功能，需在 `register_user()` 中添加：
> ```python
> if aff_code:
>     inviter = await find_user_by_aff_code(db, aff_code)
>     if inviter:
>         user.inviter_id = inviter.id
> ```

> ✅ **已修复**：`register_user()` 现已实际处理 `aff_code`：
> - 自动使用 `secrets.token_hex(8)` 为新用户生成推广码
> - 如果传入了 `aff_code`，自动查找对应邀请人并设置 `inviter_id`
> 见 `app/services/user.py` 中 `register_user()` 函数。


并在 options 表中设置：

## 十一、配置优化建议

1. **扩展 `get_system_status()`**：将 `register_enabled`、`password_register_enabled`、`email_verification_enabled`、`turnstile_check`、`github_oauth` 改为从 Option 表读取，实现运行时动态配置
2. **缓存验证码**：生产环境使用 Redis 存储邮箱验证码，支持分布式部署
3. **限制注册频率**：按 IP 限制注册请求（如每小时最多 3 次），防止滥用
4. **防止邮箱重复**：注册时检查邮箱唯一性（代码中已有 `User.email` 索引）
5. **管理后台页面化**：在前端管理页面中增加系统选项的可视化编辑界面
