# UniAPI 部署方案

## 一、部署架构

```
                    ┌──────────────┐
                    │   Client     │
                    │ (AI Agent /  │
                    │  Browser)    │
                    └──────┬───────┘
                           │ HTTPS
                    ┌──────▼───────┐
                    │   Nginx      │
                    │ (堡塔面板管理) │
                    └──────┬───────┘
                           │ 反代 :3000
                    ┌──────▼───────┐
                    │   UniAPI     │
                    │   Backend    │
                    │   :3000      │
                    └──────┬───────┘
                           │
                    ┌──────▼──────┐
                    │  SQLite     │
                    │  uniapi.db  │
                    └─────────────┘
```

**部署方式**：裸机部署（非 Docker），由 宝塔面板 管理 Nginx 反代。

## 二、环境变量配置（.env）

### 基础设置

```bash
# Server ── 注意端口 3000，与 Nginx 反代目标一致
SERVER_PORT=3000
DEBUG=false

# ── 关键：数据库 ──
# 务必使用绝对路径，避免因工作目录不同导致数据错乱
SQLITE_PATH=/www/wwwroot/api.ccbot.chat/uniapi.db

# Session 密钥（生产环境务必设置随机字符串，否则重启后所有会话失效）
SESSION_SECRET=【执行 openssl rand -hex 32 生成，粘贴到这里】
COOKIE_MAX_AGE_HOURS=168
```

### 上游 API Key（不设置全局 Key）

```bash
# ── 注意：以下所有 Key 留空 ──
# 渠道 API Key 必须在后台管理 → 渠道 → 编辑 → API Key 中单独设置
# 未设置 Key 的渠道会直接返回鉴权错误，不会兜底到全局配置
DEEPSEEK_API_KEY=
GLM_API_KEY=
QWEN_API_KEY=
KIMI_API_KEY=
MINIMAX_API_KEY=
```

### 配额与预算

```bash
# 不使用 Redis 预算功能，保持禁用
BUDGET_REDIS_URL=
BUDGET_ENABLED=false
```

### 速率限制

```bash
API_RATE_LIMIT=480
RELAY_RATE_LIMIT=480
```

## 三、Nginx 配置（堡塔面板）

### 在堡塔面板中操作

1. **宝塔面板 → 网站 → 添加站点**
   - 域名：`api.ccbot.chat`
   - 创建完成后进入"设置"

2. **设置 → 反向代理 → 添加反向代理**
   - 代理名称：`uniapi`
   - 目标 URL：`http://127.0.0.1:3000`
   - 发送域名：`$host`

3. **设置 → 配置文件**（修改 server 段，添加以下关键参数）

```nginx
server {
    listen 443 ssl http2;
    server_name api.ccbot.chat;

    # SSL 证书（堡塔面板自动管理）
    ssl_certificate    /www/server/panel/vhost/cert/api.ccbot.chat/fullchain.pem;
    ssl_certificate_key /www/server/panel/vhost/cert/api.ccbot.chat/privkey.pem;

    # ── 关键配置：SSE 流式支持 ──
    # AI 编程智能体大量使用 SSE 流式请求，以下参数必须配置
    proxy_http_version 1.1;
    proxy_set_header Connection '';
    proxy_buffering off;
    proxy_cache off;
    proxy_read_timeout 300s;   # 流式请求最长等 5 分钟（兼容长思考模型）
    proxy_send_timeout 60s;
    client_max_body_size 16m;  # 允许图片 base64 的大请求体

    # 反向代理配置
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_redirect off;
    }

    # 静态资源缓存（前端资源）
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        proxy_pass http://127.0.0.1:3000;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
```

> **堡塔面板操作说明**：在"网站 → api.ccbot.chat → 设置 → 配置文件"中，将上述 location 块内容添加到 server 块内。堡塔自带 SSL 管理和 HTTP→HTTPS 跳转。

### 关键 Nginx 参数说明

| 参数 | 推荐值 | 说明 | 在堡塔哪里设置 |
|------|--------|------|--------------|
| `proxy_buffering` | off | 禁用缓冲，SSE 实时推送 | 配置文件直接修改 |
| `proxy_read_timeout` | 300s | 长思考模型不超时 | 配置文件直接修改 |
| `proxy_http_version` | 1.1 | SSE 需要持久连接 | 配置文件直接修改 |
| `client_max_body_size` | 16m | 大请求体支持 | 配置文件直接修改 |

## 四、启动方式

### 安装依赖

```bash
cd /www/wwwroot/api.ccbot.chat
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
```

### 构建前端

```bash
cd web && yarn build
```

### 启动后端

```bash
# 前台启动（调试用）
cd /www/wwwroot/api.ccbot.chat
source .venv/bin/activate
uvicorn app.main:app --host 127.0.0.1 --port 8000

# 后台启动（推荐用 supervisor 或 screen）
nohup uvicorn app.main:app --host 127.0.0.1 --port 8000 > uniapi.log 2>&1 &
```

### 使用 Supervisor 管理（推荐）

安装 supervisor：
```bash
apt install supervisor  # Ubuntu/Debian
# 或
yum install supervisor  # CentOS
```

创建配置文件 `/etc/supervisor/conf.d/uniapi.conf`：

```ini
[program:uniapi]
directory=/www/wwwroot/api.ccbot.chat
command=/www/wwwroot/api.ccbot.chat/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
user=root
autostart=true
autorestart=true
startretries=3
stderr_logfile=/www/wwwroot/api.ccbot.chat/logs/supervisor.err.log
stdout_logfile=/www/wwwroot/api.ccbot.chat/logs/supervisor.out.log
environment=PATH="/www/wwwroot/api.ccbot.chat/.venv/bin:%(ENV_PATH)s"
```

启动：
```bash
supervisorctl reread
supervisorctl update
supervisorctl start uniapi
```

> **验证：** 部署后务必执行 `supervisorctl status` 确认进程状态。进程意外退出后 Supervisor 会自动拉起，避免服务长时间不可用。

## 五、可靠性保障

### 1. 健康检查

后端提供 `/health` 端点，返回 `{"status":"healthy","service":"uniapi-py","version":"0.10.17"}`。可用堡塔面板的"定时任务"监控：

```bash
curl -f http://127.0.0.1:3000/health || supervisorctl restart uniapi
```

### 2. Supervisor 自动重启

进程意外退出后 Supervisor 会自动拉起，最多重试 3 次。

### 3. 数据备份

堡塔面板 → 定时任务 → 添加 Shell 脚本：

```bash
#!/bin/bash
BACKUP_DIR=/www/wwwroot/api.ccbot.chat/backups
mkdir -p $BACKUP_DIR
cp /www/wwwroot/api.ccbot.chat/uniapi.db $BACKUP_DIR/uniapi-$(date +%Y%m%d).db
# 保留最近 30 天
find $BACKUP_DIR -name "uniapi-*.db" -mtime +30 -delete
```

建议每天凌晨 3 点执行。

### 4. 渠道故障自动切换

代码已内置：同一渠道连续 3 次 5xx 错误 → 自动禁用，下次请求路由到同类型其他渠道。

## 六、编程智能体调用优化

### 统一入口

AI Agent（Claude Code、Cursor、Copilot 等）直接使用 OpenAI 兼容格式：

```
Base URL: https://api.ccbot.chat/v1
API Key:  sk-xxx  （后台创建的令牌）
Models:    deepseek-v4-pro, glm-4.7-flash, kimi-k2.5, ...
```

所有请求走同一端点，UniAPI 自动路由到对应渠道。

### Token 权限管理

每个 AI Agent 使用独立 Token，限制可用模型和额度：

1. 管理后台 → 令牌 → 创建令牌
2. "允许的模型"勾选该 Agent 需要的模型
3. 设置额度上限

### 推荐模型

| 用途 | 推荐模型 | 说明 |
|------|---------|------|
| 编程主力 | deepseek-v4-pro | 代码能力强，成本低 |
| 快速响应 | deepseek-v4-flash | 低延迟简单任务 |
| 中文优化 | glm-4.7-flash / qwen3.5-flash | 中文场景更佳 |
| 复杂推理 | kimi-k2.5 | 长上下文深度推理 |

## 七、自动部署流程

前提：本地已配置 SSH 免密登录到服务器。

### 首次部署

```bash
# 1. 本地打包
cd /path/to/uniapi-py
cd web && yarn build && cd ..

# 2. 复制到服务器
rsync -avz --delete \
  --exclude '.venv' \
  --exclude '.git' \
  --exclude '__pycache__' \
  --exclude '*.pyc' \
  --exclude '.env' \
  --exclude 'uniapi.db' \
  --exclude 'web/node_modules' \
  --exclude 'web/src' \
  ./ root@api.ccbot.chat:/www/wwwroot/api.ccbot.chat/

# 3. 服务器上安装依赖
ssh root@api.ccbot.chat "cd /www/wwwroot/api.ccbot.chat && python3 -m venv .venv && source .venv/bin/activate && pip install -e ."

# 4. 配置 Supervisor（仅首次）
# 将 supervisor 配置文件复制到服务器
scp supervisor.conf root@api.ccbot.chat:/etc/supervisor/conf.d/uniapi.conf
ssh root@api.ccbot.chat "supervisorctl reread && supervisorctl update && supervisorctl start uniapi"
```

### 日常更新部署

项目根目录已创建 `deploy.sh` 一键部署脚本：

```bash
# 确保 SSH 免密登录已配置
ssh root@api.ccbot.chat "echo 'SSH OK'"

# 执行部署
bash deploy.sh
```

`deploy.sh` 自动完成：
1. 构建前端（`web/yarn build`）
2. 用 rsync 同步代码到服务器（自动排除 `.venv`、`node_modules`、`.env`、`uniapi.db` 等）
3. 服务器上安装 Python 依赖
4. 重启 Supervisor 管理的 uniAPI 服务
5. 验证健康检查和进程状态

### Supervisor 配置文件

`/etc/supervisor/conf.d/uniapi.conf`：


## 九、监控建议

### 核心指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| 上游 API 错误率 | > 5% | 渠道 Key 过期或欠费 |
| Relay 延迟 | > 10s | 上游响应慢 |
| 渠道连续失败 | = 3 | 渠道被自动禁用 |
| 磁盘使用率 | > 80% | 日志/数据库膨胀 |
| Token 配额 | < 10% | 需充值或调整 |

### 日志查看

```bash
# 实时日志
tail -f /www/wwwroot/api.ccbot.chat/logs/supervisor.out.log

# 只查看 relay 请求
grep "AUDIT" /www/wwwroot/api.ccbot.chat/logs/supervisor.out.log

# 管理后台查看测试日志
# → 日志 → 筛选类型=测试
```
